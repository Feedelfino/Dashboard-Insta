# Proposta do Projeto — Dashboard Insta Analytics

## Contexto

O Dashboard Insta Analytics é um projeto full stack criado para simular uma plataforma de análise de métricas inspirada em dados de Instagram.

A proposta surgiu como forma de praticar desenvolvimento web em um cenário próximo de uma aplicação real, envolvendo backend, frontend, autenticação, banco de dados, consumo de API e visualização de informações.

## Problema

Muitos dados de redes sociais são apresentados de forma isolada, dificultando a leitura rápida da evolução de métricas importantes.

Para quem trabalha com marketing, conteúdo ou gestão de redes sociais, acompanhar indicadores como seguidores, alcance, impressões e engajamento é essencial para tomada de decisão.

## Objetivo

Criar uma aplicação capaz de:

- autenticar usuários
- proteger rotas privadas
- armazenar e disponibilizar métricas
- exibir dados atuais em cards
- mostrar histórico de métricas em gráfico
- servir como base para uma futura plataforma de analytics mais completa

## Público-alvo

Este projeto pode ser útil para:

- criadores de conteúdo
- gestores de marketing
- analistas de redes sociais
- pequenos negócios
- profissionais que precisam acompanhar métricas digitais

## Escopo inicial

A primeira versão do projeto foca em:

- backend com FastAPI
- banco SQLite
- login com JWT
- frontend com React e TypeScript
- dashboard protegido
- visualização de métricas atuais
- gráfico de histórico de seguidores

## Escopo futuro

Possíveis evoluções:

- integração com API real do Instagram
- filtros por período
- múltiplas contas
- dashboard responsivo
- relatórios exportáveis
- alertas de crescimento ou queda de métricas
- deploy online

## Valor do Projeto

Além de funcionar como aplicação prática, o projeto também tem valor como portfólio técnico, demonstrando conhecimentos em:

- desenvolvimento backend
- desenvolvimento frontend
- autenticação
- APIs REST
- visualização de dados
- versionamento
- documentação de projeto

## Status Atual

O projeto já possui uma base funcional com autenticação, frontend integrado ao backend, dashboard protegido e visualização de métricas.

A próxima fase será melhorar a experiência visual, organizar melhor a interface e preparar o projeto para deploy e apresentação no portfólio.
